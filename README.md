# Stock Deepread

中文名：股票深读

上市公司披露、财报、电话会议、上市文件和研报的深度解读 Skill。

## 这个 Skill 解决什么问题

很多 AI 财报解读只是在“总结文本”：收入增长多少、利润下降多少、管理层说了什么。真正有价值的解读应该回答更难的问题：

- 这次披露到底变了什么？
- 变化是短期噪音、会计影响、周期波动，还是经营拐点？
- 公司在产业链哪一层，是否拥有真实议价权？
- 上下游、客户、供应商、库存、产能和替代品发生了什么？
- 股价已经大涨或大跌时，市场到底在定价什么？
- 研报、新闻和社媒观点哪些可信，哪些只是叙事？

`stock-deepread` 的目标是让 AI 自动补齐公开信源，并把披露文件翻译成一份可读、可验证、可分享的 HTML 深度报告。

## 适用场景

你可以只输入一句话，不需要提供原始文件或 URL：

```text
解读特斯拉最新财报与电话会议内容。
```

也可以指定市场、文件类型或分析重点：

```text
解读宁德时代最新年报，重点看产业链上下游。
```

```text
英伟达最新财报后为什么大跌？做一份 HTML 解读。
```

```text
解读小米集团最新业绩和最近券商观点。
```

```text
帮我解读某港股 IPO 的上市文件，重点看商业模式、财务质量和估值。
```

## 覆盖范围

| 市场 | 核心信源 |
|---|---|
| 美股 | SEC EDGAR、公司 IR、10-K、10-Q、8-K、20-F、6-K、S-1/F-1、电话会、XBRL 财务数据 |
| 港股 | HKEXnews、公司 IR、年报、中报、业绩公告、招股书、业绩会材料 |
| A 股 | 巨潮资讯、上交所、深交所、北交所、年报、季报、业绩预告、问询函、调研纪要、公告 |

多地上市公司会单独处理代码映射、主披露市场、交易时段、汇率和投资者结构差异。

## 输出内容

默认输出一份单文件 HTML 报告，包含：

- 顶部结论条：一句话结论、信心等级、最大分歧、下一催化。
- 关键数据仪表盘：股价表现、收入、利润、毛利率、现金流、估值、订单或关键经营指标。
- “这次真正变了什么”：相对上一期、相对指引、相对市场预期的变化。
- 财务桥：同比、环比、共识、指引和分部变化。
- 业务分部深读：增长、利润、风险、管理层说法和后续指标。
- 产业链地图：终端需求、系统/平台、组件、设备/材料、产能和原料。
- 电话会解读：prepared remarks、Q&A、回避点、措辞变化和管理层可信度。
- 市场反应归因：基本面 surprise、预期与仓位、估值、交易结构、社媒叙事。
- 研报与外部观点：评级变化、目标价变化、核心假设、与披露事实的冲突。
- 情景与证伪：Bull/Base/Bear、关键假设、触发条件、证伪信号。
- 信源与证据表：URL、日期、证据等级和支撑结论。

## 证据边界

| 等级 | 来源 | 用法 |
|---|---|---|
| A | 监管披露、交易所公告、公司 IR 原文、官方结构化数据 | 可作为事实底座 |
| B | 电话会 transcript、业绩会 PPT、管理层公开采访 | 可作为管理层口径，需要交叉验证 |
| C | 权威行业机构、主流财经媒体、公开研报摘要、同业公告 | 可作为背景与外部验证 |
| D | 社媒、论坛、KOL、搜索热度 | 只能作为市场情绪和叙事线索 |
| E | AI 推断、模型估算、缺失数据插补 | 必须明确标注为推断 |

社媒边界保持严格：D 级来源不能单独支撑事实结论、估值结论或买卖建议。
